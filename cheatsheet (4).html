WWW - World Wide Web
Main Developer: Tim Berners Lee
First IP-Connection: 1988 between Europe and North America

In 1990 the Hypertextproject WorldWideWeb which was a web of hypertext-documents was released
In 1993 the www was free to anyone and the Mosaic-webbrowser was released

The internet and the World Wide Web aren't the same thing!
The URL (uniform resource locator) and URI(uniform resource identifier) are a system for (globally unique) identifiers for resources on the Web

HTML is the short version of Hypertext Markup Language
HTTP is the short version of Hypertext Transfer Protocol

The URL is often confused with the URI(Uniform Resource Identifier) even thoug it's just a specific form of an URI.

The syntax can be very simple or a little bit complicated, it completely depends on the specifix URL. For example it could be scheme:path which is the simplest form.
After the scheme: you can add //host and after path "?[query]"so it would be scheme://host path?query.
After the // we can add userinfo@ and after the host you can add :port so it finally would be scheme://userinfo@host:portpath?query
Some possibles schemes are http, https and ftp.

Web Server

A webserver should store, process and deliver webpages to clients. The communication between the server and the client takes place using http. Mostly delivered are HTML-Documents.
A web browser or a web crawler starts to communicate by making a request for a specific resource using HTTP or HTTPS. The server responds with the resource.
Also content can be received from clients. This is used for web forms.

Web Client

Also known as Web Browser, a web client is an application for accessing information on the world wide web. It fetches information resources and displays them to an end device like a computer.
The process is the following: The user inputs a URL, the rendering engine of the browser renders it and then displays it.


About Doctyp


The !DOCTYPE on the head of a html document exists to show the browser the version of html that is used.


About the body element:
The body element contains all of the contents such as text, images and other things.


About the head element:


The head element contains all of the metadata.


About the html element:


The html element shows the browser that the following document is a html document. It also is the root for all of the other tags and elements of the html document.


About the b element:

The b elment is used to show important text.


About the title element:

The title element is required in a html document and it shows the name of the document in the chrome for example.


Void element:

A void element is an element whose content model never allows it to have contents.
As an example, "br" is a void element


HTML Attributes:

Attributes provide additional information about HTML elements.
As an example "style" specifies an inline CSS style for an element


Correct "!DOCTYPE" in HTML5


The correct "!DOCTYPE" in html is "!DOCTYPE html".
